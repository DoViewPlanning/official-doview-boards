# DoView® Planning Attribution and Trademark Use

It is intended to support broad, accurate, descriptive use of DoView® terminology while keeping official endorsement, certification, badge use, and affiliation clear.

## 3. Descriptive use is encouraged

You may use the DoView® marks in a fair, accurate, and descriptive way to say that your work uses, refers to, supports, implements, builds on, extends, or is compatible with DoView Planning, DoViews, DoView Boards, or related DoView-based models.

For example, you may say:

- “This project uses the DoView Planning approach.”
- “This board is based on DoView Boards.”
- “This app supports DoView Boards.”
- “This tool creates DoView-compatible boards.”
- “This platform implements the DoView Boards minimum standard.”
- “Training in the DoView Planning approach.”
- “Consulting using DoView Planning and DoView Boards.”

You may also use DoView in the name or description of your own product, service, app, platform, integration, repository, training, consulting, or other work, provided your use is accurate and does not suggest official endorsement, certification, approval, quality assurance, affiliation, or production by Dr Paul Duignan, DoView Corporation Limited, or any official DoView® body unless agreed in writing.

## 4. Recommended terminology

When referring to the approach, use the term **DoView Planning** where practical.

When referring to diagrams, boards, or models, suitable terms include:

- DoViews
- DoView Boards
- DoView Strategy/Outcomes Models
- DoView Theories of Change
- DoView Intervention Logics
- DoView Program Theories
- DoView-compatible boards
- DoView-based models

Use the term that accurately describes your work.

## 5. Recommended acknowledgments

Please acknowledge DoView Planning and Dr Paul Duignan when you use the approach or create DoViews, DoView Boards, or related DoView-based models.

A suitable general acknowledgment is:

> This work uses the DoView Planning approach and DoView Boards developed by Dr Paul Duignan. See DoViewPlanning.org.

For software, apps, platforms, integrations, AI systems, or developer tools, a suitable acknowledgment is:

> This software supports DoView Boards using the DoView Planning approach developed by Dr Paul Duignan. See DoViewPlanning.org. This software is not affiliated with or endorsed by DoView®.

The acknowledgment may appear in any reasonable and practical location, such as:

- an About page
- documentation
- help material
- licence or notice material
- an acknowledgments page
- a settings or information screen
- a README or repository notice
